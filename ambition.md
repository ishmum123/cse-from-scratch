# PROJECT_AMBITION.md

# Rebuilding Computer Science From Scratch

> Learn computer science the way humanity could have invented it.

---

# Mission

This project is not a collection of interview questions.

It is not a CLRS summary.

It is not another system design course.

It is a journey through the inventions that built modern computing.

The reader should leave not only knowing algorithms and systems, but understanding **why they had to exist.**

---

# Ultimate Goal

By the end of the curriculum, the learner should be able to design, build, and reason about systems such as:

* Git
* PostgreSQL
* Redis
* Linux Scheduler
* Raft
* Kafka
* Elasticsearch
* Kubernetes
* An LLM inference server

...because every underlying idea has already been reconstructed.

---

# Philosophy

Computer science is a history of solving constraints.

Every topic exists because something failed.

We do not teach solutions.

We teach the failures that demanded those solutions.

---

# Immutable Rules

## Rule 1

Every discovery begins with reality.

Never start with an algorithm.

Start with a problem.

---

## Rule 2

The learner should suffer the problem before seeing the solution.

Never introduce Hash Maps.

Instead ask:

> Why is searching becoming slow?

---

## Rule 3

Every discovery introduces exactly one new idea.

---

## Rule 4

Implement everything once.

Arrays.

Linked Lists.

Hash Tables.

Heaps.

Trees.

Schedulers.

Caches.

Consensus.

Everything.

---

## Rule 5

Every abstraction must already have a concrete implementation.

Libraries come last.

---

## Rule 6

Every discovery must answer

"Why couldn't the previous solution continue?"
